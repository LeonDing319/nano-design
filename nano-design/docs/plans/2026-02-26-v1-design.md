**核心架构原则：**
- `engines/` 存放纯计算逻辑，不含 React 依赖，方便测试和复用
- `components/canvas/EffectCanvas.tsx` 是唯一渲染组件，根据效果类型调用不同 engine
- 状态管理用 useReducer + Context，应用足够简单不需要额外状态库

## 3. 数据模型

```typescript
type EffectType = 'glitch' | 'ascii'

interface GlitchParams {
  stripeDensity: number      // 1-100
  displacement: number       // 0-100
  rgbSplit: number          // 0-50
  clipShape: 'circle' | 'rectangle' | 'none'
  randomSeed: number        // 0-9999
  animation: boolean
  animationSpeed: number    // 1-10
}

interface AsciiParams {
  charDensity: number       // 1-100
  charSet: 'standard' | 'minimal' | 'blocks' | 'custom'
  customChars?: string
  fontSize: number          // 4-24
  colorMode: 'bw' | 'color' | 'mono'
  monoColor?: string
  bgColor: string
  invert: boolean
}

type ExportFormat = 'png' | 'gif' | 'html' | 'canvas-code'

interface AppState {
  image: HTMLImageElement | null
  activeEffect: EffectType
  glitchParams: GlitchParams
  asciiParams: AsciiParams
  locale: 'zh' | 'en'
}
```

## 4. 渲染引擎

### 4.1 Glitch Engine

1. 将源图按 `stripeDensity` 切成 N 条水平条纹
2. 每条条纹根据 `randomSeed` 生成伪随机水平位移（幅度由 `displacement` 控制）
3. 将图像分离为 R/G/B 三通道，各通道按 `rgbSplit` 值做水平偏移后合成
4. 根据 `clipShape` 应用裁切蒙版
5. 动画模式：每帧用时间戳混入 `randomSeed` 产生变化，按 `animationSpeed` 控制刷新频率

### 4.2 ASCII Engine

1. 将源图缩放到目标字符网格尺寸（由 `charDensity` 和 `fontSize` 决定）
2. 逐像素采样亮度，映射到字符集中对应字符（亮度越高 → 笔画越密的字符）
3. 在 Canvas 上用 `fillText` 逐行逐字渲染
4. 彩色模式：取源像素颜色作为字符颜色；黑白模式：统一前景色
5. `invert`：反转亮度映射关系

### 4.3 导出引擎

| 导出类型 | 实现方式 |
|---------|---------|
| PNG | `canvas.toBlob()` → 下载 |
| GIF | `gif.js` 逐帧录制 Canvas → 合成 GIF |
| HTML 代码 | 模板字符串，将当前参数注入到自包含 HTML+JS 代码 |
| Canvas 代码 | 模板字符串，导出纯 Canvas API 的 JS 代码 |

## 6. UI 交互设计

### 布局

- **顶栏** 固定 56px：左侧 Logo "2049 Design"，中间效果 Tab（Glitch / ASCII Art），右侧语言切换 icon
- **左侧边栏** 固定 320px：上方图片上传区 + 预设选择，下方参数控制区，底部导出按钮
- **中央画布** 自适应剩余空间，图片居中显示，保持原始比例

### 关键交互

| 场景 | 行为 |
|------|------|
| 首次访问（无图片） | 画布区域显示居中上传引导：虚线边框 + 文案，左侧参数禁用（灰色） |
| 拖拽上传 | 全屏半透明遮罩 + 高亮边框，松手后渲染默认 Glitch 的 Light Glitch 预设 |
| 参数调节 | requestAnimationFrame 节流重绘 Canvas，目标 30fps+ |
| 效果切换 | 保留图片，切换参数面板和渲染引擎 |
| 预设选择 | 参数自动更新，用户可继续微调 |
| 导出 | 模态框，上方预览缩略图，下方 4 格式卡片，点击后直接处理 |
| 语言切换 | 即时切换，无需刷新 |

### 设计风格

- 极简现代：浅色背景，干净利落，类 Figma/Linear 的工具感
- 无过度装饰，重内容和效果预览
- 控件风格统一、紧凑
