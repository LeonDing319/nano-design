# 2049 Design - 产品需求文档 (PRD)

> 版本：v0.1 | 更新日期：2026-02-26 | 状态：草稿

---

## 1. 产品概述

### 1.1 产品名称

**2049 Design**

### 1.2 一句话描述

一款面向设计师和前端工程师的 Web 端图像艺术效果生成工具，用户导入图片后可通过可视化参数控制，实时生成 Glitch、ASCII Art 等风格化视觉效果，并支持导出为图片或可用代码。

### 1.3 产品愿景

让每一位设计师和前端工程师都能快速生成高质量的艺术化视觉效果，无需复杂的图形编程知识，即可将创意转化为可直接使用的设计素材或前端代码。

### 1.4 核心价值主张

| 对比维度 | 传统方式 | 2049 Design |
|---------|---------|------------|
| 制作 Glitch 效果 | 需要 Photoshop 多图层手动操作或编写 Canvas/WebGL 代码 | 导入图片 + 滑块调参，秒级生成 |
| 制作 ASCII Art | 需要专业命令行工具或手动编排字符 | 一键转换，实时预览 |
| 前端集成 | 导出图片后还需自行编写动画代码 | 直接导出可用的 HTML/CSS/JS 代码片段 |
| 参数调试 | 修改代码 -> 刷新页面 -> 反复迭代 | 滑块实时预览，所见即所得 |

## 2. 目标用户

### 2.1 用户画像

**主要用户：设计师**
- UI/UX 设计师、平面设计师、创意设计师
- 需要快速生成风格化视觉素材用于作品、社交媒体、品牌设计
- 不具备编程能力，依赖可视化工具

**主要用户：前端工程师**
- Web 前端开发者、创意开发者
- 需要在网页中嵌入艺术化视觉效果
- 希望获得可直接使用的代码片段，减少从零编写的时间成本

**次要用户：内容创作者**
- 社交媒体运营、自媒体创作者
- 需要为内容配图增加独特的视觉风格

### 2.2 使用场景

1. 设计师为科技类品牌制作赛博朋克风格的视觉素材，需要 Glitch 效果
2. 前端工程师为个人作品集网站添加动态故障效果的 Hero 图片
3. 设计师将产品截图转换为 ASCII Art 风格，用于技术博客的配图
4. 前端工程师需要一段 Glitch 动画的 Canvas 代码，嵌入到落地页中

## 3. 功能需求

### 3.1 MVP 功能范围（第一阶段）

#### 3.1.1 图片导入

| 需求项 | 描述 |
|-------|------|
| 导入方式 | 支持点击上传和拖拽上传 |
| 支持格式 | JPG、PNG、WebP |
| 文件大小限制 | 最大 10MB |
| 图片预览 | 上传后在画布区域显示原图预览 |

#### 3.1.2 效果一：Glitch 故障效果

将图片转化为水平扫描线切割 + 位移的故障艺术效果。

**可调参数：**

| 参数名称 | 控件类型 | 范围 | 说明 |
|---------|---------|------|------|
| 条纹密度 (Stripe Density) | 滑块 | 1 - 100 | 控制水平切割线的数量和精细程度 |
| 偏移强度 (Displacement Intensity) | 滑块 | 0 - 100 | 控制每条碎片水平方向的错位幅度 |
| 色彩通道分离 (RGB Split) | 滑块 | 0 - 50 | 控制 RGB 三通道的偏移距离 |
| 裁切形状 (Clip Shape) | 下拉选择 | 圆形 / 矩形 / 无裁切 | 效果的外轮廓形状 |
| 随机种子 (Random Seed) | 滑块 / 按钮 | 0 - 9999 / 随机 | 改变条纹偏移的随机分布，"随机"按钮一键刷新 |
| 动画开关 (Animation) | 开关 | 开 / 关 | 是否启用 Glitch 动态闪烁效果 |
| 动画速度 (Animation Speed) | 滑块 | 1 - 10 | 仅在动画开启时显示，控制闪烁频率 |

#### 3.1.3 效果二：ASCII Art 字符画效果

将图片转化为由字符组成的 ASCII 艺术画。

**可调参数：**

| 参数名称 | 控件类型 | 范围 | 说明 |
|---------|---------|------|------|
| 字符密度 (Character Density) | 滑块 | 1 - 100 | 控制字符的精细程度，值越大细节越丰富 |
| 字符集 (Character Set) | 下拉选择 | 标准 / 简约 / 方块 / 自定义 | 选择用于渲染的字符集合 |
| 字体大小 (Font Size) | 滑块 | 4 - 24 | 控制单个字符的显示大小 |
| 彩色模式 (Color Mode) | 下拉选择 | 黑白 / 彩色 / 单色自定义 | 字符的颜色模式 |
| 背景色 (Background Color) | 色彩选择器 | - | 背景颜色 |
| 反转 (Invert) | 开关 | 开 / 关 | 反转明暗关系 |

#### 3.1.4 导出功能

| 导出格式 | 说明 | 适用场景 |
|---------|------|---------|
| PNG | 导出当前效果的静态截图 | 设计师直接使用 |
| GIF | 导出 Glitch 动画效果（仅动画模式） | 社交媒体、展示用途 |
| HTML 代码片段 | 导出包含完整效果的 HTML + CSS + JS 代码 | 前端工程师嵌入网页 |
| Canvas 代码 | 导出基于 Canvas API 的 JS 代码 | 前端工程师二次开发 |

#### 3.1.5 国际化 (i18n)

| 需求项 | 描述 |
|-------|------|
| 支持语言 | 中文简体、English |
| 默认语言 | 根据浏览器 `navigator.language` 自动检测，中文环境默认中文，其余默认英文 |
| 手动切换 | 设置中提供全局语言切换选项 |
| 翻译范围 | 全部 UI 文案、参数标签、提示信息、导出代码注释 |

### 3.2 后续迭代功能（第二阶段）

| 效果名称 | 描述 |
|---------|------|
| 像素化 (Pixelation) | 将图片转化为像素风格，可调节像素块大小 |
| 半色调 (Halftone) | 将图片转化为半色调网点效果，模拟印刷风格 |
| 粒子化 (Particle) | 将图片分解为粒子系统，支持动态扩散动画 |
| 波浪扭曲 (Wave Distortion) | 对图片施加波浪形变形效果 |

## 4. 页面结构与交互设计

### 4.1 整体布局

```
+----------------------------------------------------------+
|  Logo(2049 Design)       效果切换 Tab        设置(语言/主题) |
+----------------------------------------------------------+
|                    |                                      |
|                    |                                      |
|    参数控制面板      |          画布预览区域                  |
|    (左侧边栏)       |        (中央主区域)                    |
|                    |                                      |
|   - 导入图片按钮     |      实时渲染效果预览                   |
|   - 效果参数滑块     |                                      |
|   - 导出按钮        |                                      |
|                    |                                      |
+----------------------------------------------------------+
```

### 4.2 用户操作流程

```
用户访问 2049 Design
       |
       v
  选择/拖拽上传图片
       |
       v
  默认展示 Glitch 效果（使用预设参数）
       |
       v
  用户通过左侧面板调节参数 <---> 画布实时更新预览
       |
       v
  用户切换效果 Tab（如切到 ASCII Art）
       |
       v
  继续调节参数，满意后点击导出
       |
       v
  选择导出格式（PNG / GIF / Code）
       |
       v
  下载文件或复制代码到剪贴板
```

### 4.3 关键交互细节

| 交互场景 | 行为描述 |
|---------|---------|
| 首次访问（无图片） | 显示引导区域：居中的上传提示 + 示例效果展示 |
| 拖拽上传 | 全屏拖拽热区，拖入时显示高亮边框反馈 |
| 参数调节 | 滑块拖动时画布实时更新，需确保渲染性能流畅（目标 30fps+） |
| 效果切换 | 切换 Tab 时保留已上传的图片，仅更换效果和参数面板 |
| 导出代码 | 弹出代码预览弹窗，支持一键复制，代码包含使用说明注释 |
| 语言切换 | 即时切换，无需刷新页面 |

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 目标值 |
|------|-------|
| 首屏加载时间 | < 3 秒（全球 CDN） |
| 图片上传响应 | < 1 秒（10MB 以内） |
| 效果渲染帧率 | >= 30fps（参数调节时实时预览） |
| 导出处理时间 | PNG < 2 秒，GIF < 10 秒，代码 < 1 秒 |

### 5.2 兼容性

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| 设备 | 桌面端为主，最小分辨率 1280x720 |
| 移动端 | 暂不支持，后续版本考虑响应式适配 |

### 5.3 安全与隐私

- 所有图片处理均在客户端（浏览器端）完成，**不上传到服务器**
- 无用户数据收集，无 Cookie 追踪
- 站点支持 HTTPS

## 6. 技术方向建议

> 以下为技术方向建议，具体技术选型在技术设计阶段确定。

| 层面 | 建议方向 |
|------|---------|
| 前端框架 | React / Next.js（SSG 模式，适合纯前端应用） |
| 图像处理 | Canvas API + WebGL（性能敏感的效果渲染） |
| 动画 | requestAnimationFrame + Canvas 逐帧渲染 |
| 国际化 | i18next 或 next-intl |
| 代码导出 | 模板引擎生成可独立运行的代码片段 |
| 部署 | Vercel / Cloudflare Pages（全球 CDN） |
| 分析 | 轻量匿名统计（如 Plausible），尊重隐私 |

## 7. MVP 里程碑规划

| 阶段 | 内容 | 预估周期 |
|------|------|---------|
| M1：基础框架 | 项目初始化、页面布局、图片上传、国际化框架 | 1 周 |
| M2：Glitch 效果 | Glitch 效果核心渲染引擎、全部可调参数、动画模式 | 1.5 周 |
| M3：ASCII Art 效果 | ASCII Art 渲染引擎、全部可调参数 | 1 周 |
| M4：导出系统 | PNG 导出、GIF 导出、代码片段导出 | 1 周 |
| M5：打磨上线 | UI 打磨、性能优化、浏览器兼容测试、部署上线 | 1 周 |
| **合计** | **MVP 完整版本** | **约 5.5 周** |
